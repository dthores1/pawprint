# Rescue Sites — Test Cases

Manual test cases for the **Rescue Sites** area (`/sites`) — reported locations
such as colonies, pickups, and trapping sites. Each case is one path with steps
and expected results.

## Roles

- **Manager** — owner/admin, or a member with the **MANAGE_SITES** permission.
  May create / edit / delete sites, add & remove volunteers, and add animals from
  a site.
- **Member** — any signed-in user. Reads sites, can **join/leave** a site and
  **add notes**, but sees no create/edit/delete controls.

## Conventions

- Navigate to **Rescue Sites** (`/sites`). The list has tabs **New / Nearby /
  My Sites / All Sites**. A site's detail is at `/sites/:id`.
- **Statuses:** Reported → Assessing → Active → Monitoring → Closed (lifecycle
  labels only; not a strict flow — any status can be set on edit).
- **No notifications** are generated by anything in this area (create/edit/delete,
  join/leave, volunteers, notes are all silent).
- Unless stated, "you" = the signed-in user.

## 1. Browsing & finding sites

### RS-01 — List tabs
1. Open **Rescue Sites** and cycle the tabs.
- **Expected:** **New** = sites created in the last 7 days (newest first);
  **Nearby** = sites with coordinates, nearest first; **My Sites** = sites you
  lead or volunteer at (A–Z); **All Sites** = every site (A–Z). Cards show name,
  address, status pill, and (when known) distance + contact.

### RS-02 — Distance sorting & fallback
1. Open **Nearby** with location available (allow the browser prompt, or have a
   profile address).
2. Then open it with **no** location.
- **Expected:** With location, sites sort nearest-first and cards show
  "X miles away (straight-line distance)". Without it, a hint appears
  ("Enable location access or add an address…") and all sites show instead.

### RS-03 — Empty states
1. View a tab with no matching sites (e.g. **New** with nothing added recently).
- **Expected:** Per-tab empty copy — "No new sites", "No sites assigned to you"
  (My Sites), or "No sites yet" (All).

### RS-04 — Create control is Manager-only
1. As a **Member**, view the list.
- **Expected:** No **New Site** button. (A Manager sees it.)

---

## 2. Create & edit (Manager)

### RS-05 — Create a site
1. **New Site**; enter a **Name** (required), pick a **Status**, and start typing
   an **Address** to select a Google Places match. Optionally set **Point of
   contact**, **Site Lead**, and **Notes**; save.
- **Expected:** Site created (default status **Reported** if unchanged); the
  chosen address resolves to coordinates so the map + distance work. Site Lead
  defaults to **you** and appears atop Site Volunteers.

### RS-06 — Name required
1. Open **New Site**, leave **Name** blank, try to save.
- **Expected:** Validation blocks submit until a name is entered.

### RS-07 — Free-text address (no match)
1. Type an address that returns no Places match and enter it as plain text; save.
- **Expected:** Saves with the raw address; with no coordinates the map shows
  "No location set" and the site won't sort under **Nearby**.

### RS-08 — Edit a site
1. On a site's detail, **Edit**; change status/contact/notes/address; save.
- **Expected:** Changes persist; the header status pill + description update.

### RS-09 — "+ New Contact" inline
1. In **Point of contact** (create or edit), use **+ New Contact**.
- **Expected:** A create-contact flow opens; the new person becomes the selected
  contact. (Support/system accounts, e.g. `@whiskerville.app`, never appear in
  this picker.)

## 3. Linked animals

### RS-10 — Site Summary counts
1. Open a site with linked animals of mixed statuses.
- **Expected:** **All** = total linked; **Released**, **Adopted**, and **Still in
  Care** (everything not released/adopted/deceased) tally correctly.

### RS-11 — Add an animal from the site (Manager)
1. In the **Animals** card, **+ Animal**; fill the form and save.
- **Expected:** The animal is created **linked to this site** and its **Intake
  Source defaults to "Rescue Site"** (editable). It appears in the site's Animals
  list and Summary.

### RS-12 — Add a litter from the site (Manager)
1. **+ Litter**; add members; save.
- **Expected:** The litter's animals are created linked to the site (same intake
  default).

### RS-13 — Animals controls are Manager-only
1. As a **Member**, view a site with animals.
- **Expected:** The list is visible but there are no **+ Animal / + Litter**
  buttons. Empty state reads "No animals linked to this site yet."

---

## 4. Volunteers & join/leave

### RS-14 — Join a site (Member)
1. As a Member who isn't on the roster, click **Join Site**.
- **Expected:** You're added to **Site Volunteers**; the button becomes **Leave
  Site**. No notification.

### RS-15 — Leave a site
1. As a listed volunteer (not the lead), click **Leave Site**.
- **Expected:** You're removed from the roster; the button returns to **Join Site**.

### RS-16 — Site Lead can't leave
1. View a site where **you are the Site Lead**.
- **Expected:** Header shows "You're volunteering here" (no Leave button); the lead
  is pinned to the top of Site Volunteers with a **Site Lead** badge.

### RS-17 — Add a volunteer with a role (Manager)
1. In **Site Volunteers**, **+ Add**; pick a person, optionally type a role
   ("Feeder", "Trapper"); confirm.
- **Expected:** They appear on the roster with the role. People already on the
  roster (and the lead) are excluded from the picker.

### RS-18 — Remove a volunteer (Manager)
1. Click the **✕** next to a volunteer.
- **Expected:** They're removed from the roster.

### RS-19 — Volunteer management is Manager-only
1. As a **Member**, view Site Volunteers.
- **Expected:** No **+ Add** or remove (✕) controls; roster is read-only (Join/Leave
  still available via the header).

---

## 5. Notes

### RS-20 — Add a note (any Member)
1. In the **Notes** card, click **+ Note**, type, **Add Note**.
- **Expected:** The composer is hidden until **+ Note** is clicked; after saving,
  the note appears newest-first, attributed to you with a date, and the composer
  collapses.

### RS-21 — Notes empty state
1. Open a site with no notes.
- **Expected:** Icon + "No notes yet." with the hint about keeping the team in the
  loop; the **+ Note** button is present.

---

## 6. Location / map

### RS-22 — Pin opens a card, then Google Maps
1. On a site with coordinates, tap the map **pin**; tap it **again**.
- **Expected:** First tap opens a small card (site name + **Open in Google Maps
  ↗**); a second tap (or the link) opens Google Maps at the site's **address**
  (not raw coordinates), matching the Details address link. Nearby Google place
  labels aren't clickable.

### RS-23 — Map fallback
1. Open a site with no coordinates (free-text address).
- **Expected:** A placeholder ("No location set for this site") instead of a map.

---

## 7. Delete (Manager)

### RS-24 — Delete a site
1. On a site's detail, **Delete**; read the confirm dialog; confirm.
- **Expected:** Dialog warns that the **N linked animals will be unlinked, not
  deleted**. After confirming, the site is removed from the list; its animals
  remain but are no longer linked to it (their site link is cleared).

### RS-25 — Delete is Manager-only
1. As a **Member**, view a site.
- **Expected:** No **Delete** (or **Edit**) button.

---

## Cross-cutting checks

- **Permission gating:** Members can read everything and use Join/Leave + Notes,
  but never see New Site / Edit / Delete / + Animal / + Litter / add-remove
  volunteer. The server (RLS) enforces this regardless of the UI.
- **No notifications:** nothing in Rescue Sites produces a notification.
- **"View as" (read-only):** while viewing as another member, site write actions
  (create/edit/delete, join/leave, volunteers, notes) are disabled.
- **Distance source:** uses the browser location if granted, else your profile
  address; it's a straight-line estimate, not a driving route.
